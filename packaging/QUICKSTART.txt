MultiCAT 0.1.0-alpha — Quick Start
==================================

One radio, many apps. MultiCAT owns your radio's CAT port and shares it with
as many programs as you like. Pre-alpha: expect rough edges. Windows x64 only.
No .NET install needed — everything is bundled.

WHAT'S IN THIS FOLDER
  Start MultiCAT.cmd        Starts the background service, then the app window.
  MultiCat.Service\         The multiplexer service (owns the radio, runs endpoints).
  MultiCat.Gui\             The MultiCAT window (status, ports, live traffic).
  MultiCat.OmniRig\         Optional OmniRig-compatible COM server + register scripts.
  appsettings.example-real-radio.json   Template for connecting a real radio.


1. TRY IT WITH THE BUILT-IN SIMULATED RADIO
   Double-click "Start MultiCAT.cmd". The window opens showing a simulated
   Elecraft K3 whose frequency drifts on its own. This needs no hardware and
   confirms everything runs on your machine.


2. CONNECT A REAL RADIO (from the app — no file editing)
   In the MultiCAT window: use "Add radio" (or Edit an existing one).
     - Serial radio: pick Connection = Serial, choose the COM port and baud,
       or pick your rig from the hamlib list to prefill those.
     - Network radio (e.g. Elecraft K4/K4D): pick Connection = Network, enter
       the radio's IP address (or K4-SNxxxx.local) and TCP port (K4 = 9200).
     - Simulator: tick the Simulator box to test with no hardware.
   Tick which network endpoints share it (rigctld for WSJT-X/fldigi, raw TCP),
   then Save. The radio connects and its live frequency appears in the status
   line. Everything is saved automatically; no restart needed.

   (You can still hand-edit MultiCat.Service\appsettings.json if you prefer —
   see appsettings.example-real-radio.json for serial and network examples.)


3. POINT YOUR SOFTWARE AT MULTICAT (no COM driver needed)
   MultiCAT exposes your radio over the network so multiple apps share it:

   WSJT-X / JTDX / fldigi / GridTracker / any hamlib app:
     Rig: "Hamlib NET rigctl"
     Network Server: 127.0.0.1:4532

   Log4OM / CW Skimmer / SDR Console / other OmniRig apps:
     Run "MultiCat.OmniRig\Register OmniRig.cmd" once (accept the prompt),
     then choose OmniRig / "Rig 1" in the app.

   Anything that speaks raw CAT over TCP: 127.0.0.1:4533

   The Traffic monitor in the MultiCAT window shows every app's requests as
   they happen, so you can see the sharing working.


KNOWN LIMITS IN THIS PRE-ALPHA
  - Radios are configured by editing appsettings.json (no GUI editor yet).
  - Real virtual COM ports (for apps like N1MM+ that only speak serial) need a
    signed com0com/VSP driver, which current Windows 11 builds reject. The
    network endpoints above are the recommended path and need no driver.
  - The OmniRig server works with most OmniRig apps; WSJT-X's OmniRig backend
    is not yet supported (use the rigctld path above for WSJT-X).
